## Codebase Patterns

### ProviderType shared schema
- **Where**: `packages/workflows/src/schemas/provider.ts`
- **Pattern**: Provider enum extracted to its own file to avoid circular imports between workflow.ts and dag-node.ts. Re-exported from schemas/index.ts and core/types/index.ts.
- **Example**: `import type { ProviderType } from './schemas/provider';`

### Config type widening pattern
- **Where**: `packages/core/src/config/config-types.ts`, `packages/workflows/src/deps.ts`
- **Pattern**: All `'claude' | 'codex'` union types replaced with `ProviderType`. New provider defaults interfaces added. WorkflowConfig.assistants must include all 4 providers as required keys.
- **Example**: `assistant: ProviderType` instead of `assistant: 'claude' | 'codex'`

### Tool implementation signature
- **Where**: `packages/core/src/clients/tools/*.ts`
- **Pattern**: All tools export `async function xxxTool(params: Record<string, unknown>, cwd: string): Promise<string>`. Path validation via shared `path-validation.ts` using `validatePath(filePath, cwd)`.
- **Example**: `import { readTool } from './tools/read';`

### Pre-existing test failures
- `packages/core/src/orchestrator/prompt-builder-knowledge.test.ts`: "truncates logs that exceed token budget" — fails on main branch
- `packages/cli/src/commands/workflow-run.test.ts`: 6 workflowRunCommand tests fail on main branch
